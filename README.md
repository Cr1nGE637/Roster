Roster — консольное приложение для работы со справочником сотрудников.

Roster — консольное приложение, реализующее работу со справочником сотрудников. Программа подключается к базе данных и поддерживает несколько режимов запуска через параметры командной строки. Все режимы соответствуют пунктам технического задания.

Во время разработки была использована слоистая архитектура: API, Application, Domain, Infrastructure. Api - входной слой, обрабатывающий режимы работы, Application - логика работы в разных режимах, Domain - модель и валидация входных данных. Infrastructure - работа с базой данных.

Для 6 пункта была проведена оптимизация запроса в бд путем создания составного индекса (Gender, FullName) с использованием операторного класса text_pattern_ops и дополнительного поля (DateOfBirth):
1) Составной индекс с text_pattern_ops строит бинарное дерево по первому символу строки, что позволяет избежать перебора строк.
2) Т.к. считывается вся таблица, включение дополнительного поля с DateOfBirth позволяет не обращаться к таблице, в следствие чего чтение данных идет только из индекса.

Дла запуска приложения необходимо ввести такие команды:
1) docker-compose up -d
2) cd ./Roster.API/
3) dotnet run -- 1

Режимы работы:
1) Создание таблицы
2) Создание записи в базе данных
3) Вывод всех сотрудников, упорядоченных по ФИО
4) Генерация 1 000 000 сотрудников
5) Замер времени для чтения из базы данных сотрудников мужского пола, чье ФИО начинает на "F"
